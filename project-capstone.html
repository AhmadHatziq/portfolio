Decision Support System with Virtual Factory Simulation

As part of the module IE3100M: System Design Project, I, along with 3 other team members, were tasked with automating the workflow for the usage for a Flexsim Simulation Model.

The purpose of the project is to help make the usage of the model easier for non-technical staff. This project can be broken into 3 main components:

  1. Extracting and cleaning data from the servers
  2. Run the simulation through the Java application
  3. Display the results of the simulation runs using Tableau dashboards

Project Details

Data Extraction

The data extraction is done via APF RTD. Data cleaning, such as string parsing and manipulation, is done. The data is then compiled into a single Excel Worksheet. This is as the Simulation Model only accepts Excel files as inputs.

Java Application to Automate the Simulation Runs

The application will propmpt the user for the specified simulation parameters. The application will confirm the user before the simulation executes. This is as each simulation run can take approximately 30 - 45 minutes.

The Java application uses a Model-View-Controller design pattern. The relevant architecture diagram, sequence diagrams and activity diagrams are here.

Data Visualization via Tableau Dashboards

After the simulation runs are completed, the data is presented by a Tableau Dashboard. The relevant KPIs are Cycle Time, OEE, Idle Rates, Throughput etc.
